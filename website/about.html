What is PolyDB?

PolyDB is a software package that converts genotype calls stored in VCF files into a PostgreSQL database and a custom made web site, allowing the easy exploration of genetic variatons by users who are not familiar with tools to query the original VCF files.

PolyDB generated web sites offer tools to filter variant calls based on genomic position, gene affected by the variation, genotype call quality (e.g. read depth, strand bias) and implement innovative ways to compare variants from multiple VCF files.
PolyDB can also be integrated with Genomeview and JBrowse genome browsers allowing the inspection of the genomic context of each polymorphic site.

Required input

PolyDB main input is one or more VCF files containing the polymorphic sites of sample genomes when compared to a single reference genome.
Optionally the user can also provide:
  • a GFF file (General feature format) describing the structural and functional annotation of genes in the reference genome. This additional file will enable the use of filters based on gene locus, gene annotation and the effect of the variation on the coding sequence of a gene: synonymous, non-synonymous, premature STOP codon, etc. .

  • the nucleotide sequence of the reference genome in FASTA format and BAM files reporting the read alignment used in the genotype calling. These files will enact the visualization of the genomic context of each variation through a genome browser (JBrowse and Genomeview) and the inspection of reads supporting each call.
